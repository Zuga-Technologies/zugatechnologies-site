# studio-cloud Profile

## Persona

ZugaCloud users are managing infrastructure — file storage, object buckets, cloud configurations, and deployment targets. Sessions are operational: they come in to upload, organize, or troubleshoot, not to explore. The mental model is close to a file manager or object storage console, not a creative tool. The UI needs to be legible, hierarchy-clear, and unobtrusive about its brand presence — the task is the focus.

## Accent rationale

ZugaCloud is the cloud family and uses sky. Sky is a lighter, more expansive hue than ZugaCode's blue — it references the literal cloud metaphor and conveys openness and scale. The sky/blue split between ZugaCloud and ZugaCode is intentional: both are technical, but cloud infrastructure (horizontal scale, buckets) reads differently from developer tooling (code review, agent tasks). See `acc10` in §5.
